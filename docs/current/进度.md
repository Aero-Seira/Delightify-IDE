- 新增 `script-workspace:*` IPC：
  - `script-workspace:list`：列出 `kubejs/server_scripts`、`client_scripts`、`startup_scripts`、`assets` 下的 `.js/.json` 文件。
  - `script-workspace:read`：读取指定相对路径文件。
  - `script-workspace:save`：只允许保存 Delightify managed 文件。
  - `script-workspace:create-managed`：在没有导出产物时创建 `kubejs/server_scripts/zzz_delightify_manual.js` 受管手动脚本，用于测试和专业用户手动入口。
- main 进程新增 `script-workspace` 服务，保存保护规则：
  - 通过 `kubejs/.delightify-generated.json` manifest 或 `@delightify-generated` marker 判定 managed。
  - user scripts 默认只读。
  - manifest 只读。
  - 受管 `.js` 保存后仍必须包含 `@delightify-generated` marker。
- 当前边界：暂不提供“接管用户脚本”；受管手动脚本不写入 generated manifest，避免 Action Workbench 后续导出清理时误删。

## 2026-06-17 · M2 U3 Action Workbench 起步

- 新增 `ActionWorkbench` 页面与 `/actions` 路由，侧边栏增加“动作工作台”入口。
- 工作台首批接入 4 个基础动作：
  - `replace`：输入 from/to item 或 tag、scope、配方类型/mod 过滤。
  - `retag`：输入 tag、add/remove、物品列表。
  - `remove_recipe`：输入配方 id 列表。
  - `rename`：输入 item、locale、新显示名。
- 页面已串接既有后端管线：
  - `engine:dry-run` 计算 operations、risk、blast 和 deferred。
  - 用户可勾选待确认 operation id 后重新 dry-run。
  - 新增 `export:kubejs:preview`，dry-run 后可预览将写出的 KubeJS fileset，不落盘。
  - `export:kubejs` 导出当前 `changeSetPreview`。
  - `export:kubejs:revert` 撤销 Delightify 受管生成文件。
- U4 生成产物预览同步起步：页面展示相对路径、operation count 和生成内容；实际写入仍受 KubeJS emitter 的 Delightify managed 覆盖保护约束。
- 当前边界：仍不接 LLM/Agent；输出层沿用现有 KubeJS emitter 和受管文件覆盖保护；输出替换等后端标记为不可确认的强风险操作仍只展示分析结果。

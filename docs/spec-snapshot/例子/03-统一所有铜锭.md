# 例子 03：统一所有铜锭（unify）

> **Claude 拟稿（待验证）**。若不符合你的设想，直接改或告诉我。

## 0. 这个例子想验证什么

`unify` 动词的完整行为：同名物品归一——但要能辨别**"名字像、实则不同"的陷阱**（铜合金锭、附魔铜锭不该并入），且**"保留哪一种"是你的判断（c）**，Agent 只给默认 + 搁置，不替你拍板。

## 1. 用户目标

> "把整合包里所有的铜锭统一成一种"

风格档案：科技向大包，多个 mod 各自加了铜锭。

## 2. 整合包上下文（id 仅示意）

### 相关物品
| id（显示名） | 语义 |
|---|---|
| `minecraft:copper_ingot`（铜锭） | 原版，纯铜 |
| `thermal:copper_ingot`（铜锭） | Thermal，纯铜 |
| `immersiveengineering:ingot_copper`（铜锭） | IE，纯铜 |
| `mekanism:ingot_copper`（铜锭） | Mek，纯铜 |
| 🪤 `alloymod:copper_alloy_ingot`（铜合金锭） | **含锡，非纯铜** |
| 🪤 `magicmod:enchanted_copper_ingot`（附魔铜锭） | **魔法变体，专用于魔法配方** |

### 相关 tag
- `#forge:ingots/copper`：含原版 / thermal / ie / mek 四种纯铜锭（**不含**合金锭、附魔铜锭）

## 3. 期望的 Agent 行为（逐案）

| # | 对象 | 分类 | 决策 | 理由 | 置信·风险 | 处置 |
|---|---|---|---|---|---|---|
| 0 | 首选项选哪个 | 选型（作者判断） | 默认 `minecraft:copper_ingot`（最通用），但这是你的判断 | "保留哪个"属设计判断 (c) | —·— | ⚠️ 搁置（头号待裁决） |
| 1 | 4 种纯铜锭（#forge:ingots/copper） | 同类物品 | **unify**：其余配方引用改指 `#forge:ingots/copper`，重复锭隐藏 | 同 tag、同语义"纯铜锭" | 高·中 | ⚠️ 搁置（列影响面） |
| 2 | 🪤 铜合金锭 | 形近异物 | **不并入** | 含锡、非纯铜、不在该 tag | 高·— | ⏭️ 跳过 |
| 3 | 🪤 附魔铜锭 | 形近异物·魔法 | **不并入** | 魔法专用变体，统一会破坏魔法配方 | 中高·— | ⏭️ 跳过（存疑则搁置） |
| 4 | 引用各 mod 铜锭的配方 | 引用改写 | 输入/输出改指 `#forge:ingots/copper` 或首选项 | 统一后须重定向引用，避免配方失效 | 高·中 | ✅ 自动（随 unify） |

## 4. 期望进入"待裁决队列"的项

- **首选项保留哪个铜锭**（默认原版）。
- **附魔铜锭是否算同类**（默认不算）。
- **重复锭：隐藏 vs 移除**（默认隐藏；移除爆炸半径高）。


## 5. 边界 / 不该做的事（反例）

- **不要**并入铜合金锭 / 附魔铜锭——名字像 ≠ 同物。（批注：但在实际生成配方时，如果目标产物的语义上能够接受相近物品，可以交给用户抉择）
- **不要**静默移除任何铜锭物品；先重定向引用，且默认只隐藏。
- **不要**改动某 mod 明确按 id 索取自家铜锭的机器配方（若存在则搁置）。

## 6. 备注 / 开放问题

- "纯铜锭"判定主要靠 `#forge:ingots/copper` 成员 + 名称/语义；合金 / 附魔靠语义排除。
- 经典 Almost Unified 场景，可作为与 AU 输出对比的基准。
- **批注：另一种路径——根据物品配方链上下文，做出类似于（低纯铜，高纯铜）的区分规划**

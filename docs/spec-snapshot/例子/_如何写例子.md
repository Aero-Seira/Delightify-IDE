# 如何写例子

## 这个目录是干什么的

例子 = **用具体场景，描述"给定整合包内容 + 用户目标，Agent 应该怎么做、为什么"**。

它有三重身份：
1. **设计驱动**：你写例子 → 我读例子 → 据此打磨 [[01-核心设计与决策模型|决策模型]] 的细节（分类法、置信信号、搁置条件…）。
2. **行为规格**：例子就是 Agent 的目标行为说明书。
3. **测试集**：将来可以直接拿例子来验证 Agent 是否做对。

## 工作方式

> 你直接在本目录新建/编辑例子文件 → 我读懂后，把其中暴露的设计点反馈进核心设计文档，必要时回来和你确认。

**精修步骤（标准流程）**：你写完粗稿后，我会把它**精修成规范版**——顶部加 §0「我理解的你的意图」一句话确认区、补全空缺列、规范记号、标注链中角色；**凡我推断/补充处标【补】请你重点核对**；你的**原始草稿完整保留在文末附录**。精修版即是我对你意图的理解，供你一眼确认或纠偏。范例见 [[02-炸物食品合成链优化]]。

你**不需要把每个例子写得很完整**。哪怕只写「目标 + 几个棘手的 case + 你期望的处理」，对设计也极有价值。低门槛 = 多写 = 设计更准。

## 怎么写

- 复制 [[_模板]] 新建一个文件，命名 `NN-场景名.md`（如 `02-统一所有铜锭.md`）。
- 按模板的小节填；**带"可选"的小节可以留空或删掉**。
- 重点写 **Agent 容易判断错、或你拿不准的地方**——那才是设计的难点。

## 书写约定（不必严格，我能理解大意）

| 写什么 | 怎么写 | 例 |
|---|---|---|
| 物品 | `id（显示名）` | `create:dough（面团）` |
| tag | `#命名空间:路径` | `#forge:dough` |
| 配方 | `输入 --[方式/机器]--> 输出` | `生面条 --[锅煮]--> 熟面条` |
| 期望动作 | 自然语言即可 | "把小麦换成面粉" |
| 处置 | `自动` / `搁置` / `跳过` | |
| 置信·风险 | `高/中/低` · `低/中/高` | `高·低` |

## 一个好例子的特征

- **包含"反例"**：明确写出 Agent **不该**做什么（最有价值的信息之一）。
- **包含"陷阱"**：表面相似但应区别对待的 case（如"煮面用生面条，而非面粉"）。
- **包含"该搁置的"**：你自己都得想一下、或有多个合理选项的情形。
- **写清"为什么"**：你凭什么判断该这么改——这正是 Agent 要学会的语义线索。

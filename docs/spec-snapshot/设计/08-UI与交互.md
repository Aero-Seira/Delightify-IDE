# UI 与交互 —— v0 草案

> **状态**：v0 草案。把抽象结构（决策清单 / 待裁决队列 / 决策支持视图 / diff）落成作者**真正看到的界面**，并映射到现有 Delightify 骨架可复用的部分。
> 目标用户 = 魔改老手、痛点 = 认知负担 → UI 的天职是**消除"查"和"看不过来"**。

---

## 1. 设计原则

1. **材质是第一语言**：凡列物品/配方处都带图标，视觉识别优先于读 id。
2. **两道闸门即两块主界面**：闸门1 = 决策清单审阅；闸门2 = diff 预览。其余面板服务于此。
3. **可扫读 + 可逐条干预**：清单默认按"需关注程度"排序，自动项可一眼略过、搁置项突出。
4. **判断交还作者**：凡 (c) 判断（首选项/范围/幅度/命名）都以"默认+可改"的控件出现，不藏在后台。

---

## 2. 主要界面

| 界面 | 作用 | 关键元素 |
|---|---|---|
| **任务输入** | 作者下达目标 | 自然语言框 + 目标模式勾选(可选捷径) + 风格档案 + 目标包选择 |
| **目录浏览器** | 反 Ctrl+F | 带图标的物品/配方列表，按 mod/tag/类型筛选、搜索；选择器结果实时"匹配 N 项"预览 |
| **配方/依赖查看器** | 看链 | 输入→输出图示、上下游链、引用者列表（爆炸半径可视化） |
| **决策清单审阅（闸门1）** | 审 Agent 的逐案决策 | 每行：`图标｜对象｜分类标签｜动作｜理由｜置信·风险｜处置`；行内 **接受/改/拒**；顶部"目标解释假设"首条确认 |
| **待裁决队列** | 集中处理搁置 | 每项：情形 + 选项卡（带效果预览）+ Agent 推荐 + 为何不确定；任务其余完成后呈现 |
| **diff 预览（闸门2）** | 确认效果 | 逐配方/逐文件 before→after；可整体/逐条应用 |
| **决策支持视图** | 引导式规划用 | [[05-引导式规划模式|设计/05 §3]] 的只读四块（口径/数据/焦点排序/旋钮）；**只读、不改数据** |

---

## 3. 流转（界面如何串起来）

```
任务输入 → (分流) ┬─ 执行型 → 决策清单审阅[闸门1] → diff 预览[闸门2] → 应用 → 产物
                  └─ 引导型 → 决策支持视图(只读) → 作者圈定/设参 → 回到执行型流程
目录浏览器 / 配方查看器：全程可旁开，供作者随时查证（图标+引用+链）。
```

## 4. 复用现有 Delightify 骨架

| 需求 | 复用 |
|---|---|
| 应用框架 | Electron `packages/main/src/{main.ts,preload.ts}` + `packages/renderer`（React） |
| 前后端通信 | `packages/renderer/src/ipc` + preload 桥 + `packages/main/src/ipc` |
| 状态管理 | `packages/renderer/src/store` |
| 组件/样式 | `packages/renderer/src/{components,theme,styles}`（CSS modules） |
| 代码/YAML 编辑 | Monaco——用于查看/手改生成的 KubeJS 脚本、Intent Spec 的 YAML 视图 |
| 中文界面 | `packages/renderer/src/i18n`（产品 UI 为中文） |
| 本地数据 | `packages/main` 的 `@libsql/client`（[[06-数据层]]） |

> 即：决策清单/队列/diff 这些**新面板**是新增 React 组件 + 新 IPC 通道；目录/配方查看器可在现有 list/tree 组件上扩展；脚本/Spec 文本编辑直接用已集成的 Monaco。

## 5. 走查（例子01 的决策清单长什么样）

```
任务：优化整合包内所有面食的配方        [目标模式：语义贴切 ✓ 去重统一 ✓ ·可改]
⓪ 我把"优化"理解为：用语义最贴切中间产物+合并重复物品  [确认] [改]

决策清单（4 自动 / 2 待裁决）
✅ [🍞小麦→面粉] 制作面团步骤 · 高·低 · replace          [接受][改][拒]
✅ [🍜面粉→生面条] 煮面步骤 · 高·低 · replace(面种见↓)    [接受][改][拒]
✅ [🥟面团→发酵面团] 蒸馒头 · 中高·低 · replace           [接受][改][拒]
⏭️ [✨魔法转化→造物面包] 域外·跳过
待裁决（2）
⚠️ 煮面用哪种生面条？  ◉通用 ○拉面 ○乌冬   推荐:通用(上游=面团擀制)
⚠️ create:dough 替换后冗余，是否清理？ ◉去引用 ○移除 ○不动  推荐:去引用(删物品风险高)
                                                  [全部应用 → 预览 diff]
```
→ 抽象的 Intent Spec 能直观落成"扫一眼+点几下"的界面，命中"消除认知负担"。✅

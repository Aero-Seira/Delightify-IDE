# LLM / Agent 集成 —— v0 草案

> **状态**：v0 草案。定义 Agent **如何被驱动**去产出一份合法的 [[03-Intent-Spec契约|Intent Spec]]（或引导式规划的决策支持视图），以及"智能"如何被安全纪律框住。
> 纪律见 [[01-核心设计与决策模型|设计/01 §6]]；动作语义见 [[04-引擎与动作语义]]；数据/图谱见 [[06-数据层]]。

---

## 1. 总原则

- LLM 只做 **(a) 语义辨识 + (b) 执行规划**，**不做 (c) 设计判断**（[[01-核心设计与决策模型|设计/01 §2]]）。
- **先查图谱再提议**：任何具体 id/配方/tag 必须来自数据层查询，**禁止凭空生成**（防幻觉 id）。
- **结构化输出是硬约束**：Agent 的产物必须是符合 Intent Spec schema 的结构化对象（工具调用 / structured output 强制），不合规即重试。
- **置信不靠自报**：见 §4。

---

## 2. 主循环：召回 → 逐案 → 搁置

```
目标
 └ Stage0 分流（执行型 / 引导型，见 设计/05）
   ├ 引导型 → 产「决策支持视图」(只读信息+选项)，结束
   └ 执行型 ↓
     Stage A 召回：LLM 把目标展开成种子 → 调图谱查询工具捞全部候选（优化召回、亮明范围）
     Stage B 逐案：对每个候选 →【分类(多facet)】→【定动作+参数】→【多信号置信 + 风险】
     汇总 → Intent Spec（decisions + deferred + recall + aggregate_findings）
```

- **先分类再动作**：每条必须先产出 `classification`（[[03-Intent-Spec契约|设计/03 §4]] 的固定枚举）再选动作——分类是可审最小单元。
- Stage A/B 可并行 over 候选；选择器后四种（语义家族/等价类/品类模板）由 LLM 在召回集上**成组**，成组结果可审可改。

---

## 3. 工具集（Agent 只读地查、不直接改数据）

只读查询工具（落到 [[06-数据层]] 的 libsql/图谱）：
`search_items(name|tag|mod)`、`get_recipes(item)`、`get_refs(item|tag)`（引用者=爆炸半径）、`get_chain(item)`（上下游链）、`get_tag_members(tag)`、`get_lang(id)`、`get_texture(id)`。

> Agent 用这些**搜证**，产出 Intent Spec；**改动一律经引擎 + 闸门**，Agent 不碰数据。

---

## 4. 多信号置信（不靠 LLM 自报"我多确定"）

对每条判断收集四类信号，**按一致程度定置信**：

| 信号 | 来源 |
|---|---|
| 名称语义 | lang 显示名 |
| tag 归属 | 数据层 tag 成员 |
| 配方类型/机器 | 配方 type |
| 图谱位置 | 上下游/引用 |

- 四者**同向** → 高；**部分缺失** → 中；**相互冲突** → 低 → **自动降级进 `defer`**。
- `confidence_signals` 字段记录哪些信号支持本判断（可审、可解释）。
- 实现上：信号收集与"一致程度"打分可大部分用**确定性代码**，LLM 只在"这条信号语义上是否支持"处做判断——比纯问模型可靠。

---

## 5. 引导式规划的交互

分流判为引导型时（[[05-引导式规划模式|设计/05]]），Agent **不产决策清单**，而是：用只读工具整合相关数据 → 产出"决策支持视图"（候选界定口径 + 客观数据 + 焦点排序 + 旋钮）→ 等作者设定 (c) 参数 → 回填生成执行型 Intent Spec → 再走主循环。

---

## 6. 模型与产出约定（架构级，细节留实现）

- 推理用**最新的 Claude 模型**（结构化输出 / 工具调用）；具体型号、参数、token 预算在实现阶段定（不在本规划展开）。
- 本地嵌入（ONNX）**推迟**；v1 检索优先用词法（FTS5 + tag/名称），[[06-数据层]] 已记。
- 输出校验：Intent Spec 经 schema 校验，不合规重试；引用的 id 必须在图谱中存在，否则判为幻觉并丢弃/重查。

---

## 7. 安全纪律在本层的落点（复述+具体化）

1. 分类前置（§2）→ 可审。
2. 多信号置信（§4）→ 不静默猜。
3. 先查图谱（§1/§3）→ 不幻觉。
4. 结构化输出 + 校验（§6）→ 产物可被引擎确定性消费。
5. 改动经引擎+闸门（§3）→ Agent 永不直接写数据。

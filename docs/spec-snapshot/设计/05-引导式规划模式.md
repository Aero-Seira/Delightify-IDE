# 引导式规划模式 —— v0 草案

> **状态**：v0 草案。定义管线最前面那道**「分流」**——什么样的需求**不该**让 Agent 自行规划执行，而应转为"整合信息 + 引导作者规划"。内嵌决策 **#5 触发条件** 已于 2026-06-12 确认（标 ✅已定）。
> 来源：[[04-降低前期配方成本|例子04]] 批注。位置见 [[01-核心设计与决策模型|设计/01 §3 管线]]「分流」一支。

---

## 1. 为什么需要它

有些需求（如"前期太肝，降一档"）**几乎全是设计判断 (c)**——"哪些算前期""肝在哪""降多少""降输入还是加产出"都该由作者定。若让 Agent 硬给默认再执行，就是**替作者做了本属于他的设计决定**，违背立身分界（[[01-核心设计与决策模型|设计/01 §2]]）。

> 正确姿态：此时 Agent = **决策支持 / 信息整合器**，不是规划者。它把"看清这事需要的全部信息"摆到作者面前，由作者规划；规划成形后才走执行管线。

---

## 2. ✅已定 · 决策 #5：触发条件

**核心判据（principled）**：

> **能否把目标编译成一份具体的 Intent Spec，且其中每个设计判断 (c) 要么由作者给出、要么有「无争议的默认」？**
> - **能** → 执行型（走正常管线，把少数不确定的 (a)/(b) 项搁置）。
> - **不能 —— 编译会被迫由 Agent「伪造」(c) 判断（范围/幅度/平衡取向）** → **转引导式规划**。

**实操检测信号**（命中即倾向引导）：
1. 目标模式以**强 (c) 取向**为主（数值平衡 / 进度调整 / "难度/节奏"类），而非以 (a) 语义为主（语义贴切 / 合理性 / 一致性）。
2. **范围未指定**：选择器无法在不做 (c) 判断的情况下确定（"前期"= 哪些？）。
3. **幅度/目标未指定**：动作参数无客观默认（"降一档"= ×多少？）。
4. **影响面大**：盲目执行会大面积改动。

> 推荐如上：**1 必要 + (2 或 3) 命中 → 引导**。纯 (a) 语义类即使范围大也走执行（如"优化所有面食"——范围可由语义召回客观确定，只有少数选型搁置）。这条把"前期太肝"判为引导、把"优化所有面食"判为执行，与例04/例01 一致。

---

## 3. 产物：只读「决策支持视图」

引导式规划**不产出决策清单、不改任何数据**，而是产出一张**只读视图**，含四块（以"前期太肝"为例，见 [[04-降低前期配方成本|例子04]]）：

1. **几种候选「界定口径」** + 各覆盖什么（"前期"= 深度≤2？电力前？你手圈？）——把模糊的 (c) 概念摊成可选项。
2. **相关对象的客观数据**：成本（输入量）、依赖深度、产出等（纯事实，无判断）。
3. **疑似"焦点"排序**：按客观信号（高成本/高重复/高深度）排出最可能是作者所指的对象——**辅助注意力，不替作者下结论**。
4. **可调旋钮**：作者要设的 (c) 参数（范围、幅度、降输入 vs 加产出、取整）。

并**明确声明**：Agent 不会自己选范围/幅度。

---

## 4. 交接回执行管线

作者在视图上①圈定范围 ②设好参数 后，**那一刻**这部分才变成一份具体的可执行任务：

```
决策支持视图 + 作者的 (c) 决定  →  生成 Intent Spec（执行型）  →  正常管线（逐案/置信×风险/搁置/闸门）
```

→ 引导式规划是执行管线的**前置澄清**，不是替代；澄清完即并入主流程。

---

## 5. 与执行型的根本区别

| | 执行型 | 引导式规划 |
|---|---|---|
| Agent 角色 | 规划者（逐案决策） | 决策支持 / 信息整合器 |
| 产物 | Intent Spec（决策清单 + 待裁决） | 只读决策支持视图（信息 + 选项 + 旋钮） |
| 是否改数据 | 经闸门后改 | **完全不改**（只读） |
| (c) 判断 | 少数搁置交作者 | **全部交作者**，Agent 只摆信息 |

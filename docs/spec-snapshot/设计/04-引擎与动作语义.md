# 引擎与动作语义 —— v0 草案

> **状态**：v0 草案。定义每个动作的**确定性语义**，让引擎消费 [[03-Intent-Spec契约|Intent Spec]] 时**无需再做任何 AI 判断**。内嵌决策 **#1 v1 动作范围**、**#4 add_item 边界** 已于 2026-06-12 确认（标 ✅已定）。
> 动作清单见 [[02-动作词汇表与分类法]]；爆炸半径数据来自 [[06-数据层]]；产物落地见 [[07-输出层-KubeJS]]。

---

## 1. 总原则

- 引擎是**纯函数**：输入 = `(知识图谱快照, Intent Spec)`，输出 = `(变更集 / diff)`。不写源 JAR，只产生**覆盖产物**（datapack/KubeJS 脚本）——因此**天然可逆**（撤销 = 删除产物）。唯一不可逆的是「世界已放置方块」类影响（见 §4）。
- 所有动作以**原始基线**为参照定义（见 scale），保证**幂等**：同一 Spec 重跑结果一致、不累积。

---

## 2. ✅已定 · 决策 #1：v1 动作范围（分层）

| 层 | 动作 | v1 |
|---|---|---|
| **自动可执行·原语** | `scale` `replace_ingredient` `remove(hide/remove_recipe)` `retag` `rename` | ✅ |
| **自动可执行·复合**（由原语编排） | `unify` `harmonize` `differentiate` `constrain_inputs`(输入集校正) | ✅ |
| **仅探测+搁置**（不自动执行） | `add_bridge_recipe` `change_recipe_type` `add_item` `remove_item`(注册移除) | ⚠️ 探测+建议 |

> 推荐如上。要点：**改配方/tag/显示名可自动**（可逆、产物覆盖）；**造新内容 / 删注册物品**只探测+搁置（爆炸半径最高）。

---

## 3. 各动作的确定性语义

### 原语
- **`scale`** · params `{field: input_count|output_count|time|energy|<custom>, factor|delta, clamp{min,max}, round: floor|round|ceil}`
  以**原始基线值**计算（`新值 = clamp(round(原值 × factor))`），故幂等；不缩放守恒型 1:1 转化（由分类 `role` + 等比识别拦截，见 §5）。
- **`replace_ingredient`** · params `{from: item|tag, to: item|tag, scope: input|output|both, preserve: count|nbt}`
  在匹配配方内把 from 换 to；默认保留数量/NBT；无 from 命中则 no-op（幂等）。
- **`remove`** · 三种强度，**默认取最弱**：
  `hide_in_jei`（仅 JEI/创造不可见，最弱、可逆） < `remove_recipe`（删配方） < `remove_item`（删注册，**进 §2 探测+搁置**）。
- **`retag`** · params `{items, tag, op: add|remove}`：改 tag 归属，影响所有引用该 tag 的配方（爆炸半径见 §4）。
- **`rename`** · params `{item, locale, new_name}`：**只改 lang 显示名，不动注册 id** → 低风险、可逆、不破坏任何引用。用于消歧。

### 复合（引擎展开为原语，且**展开过程对作者透明**）
- **`unify(group, canonical, policy)`** = `retag`(group→公共 tag) + `replace_ingredient`(其余→canonical/tag) + `hide`(重复项，可选) + 处理产出重复项的配方(redirect/remove，按 policy)。
- **`harmonize(outliers, family)`** = `replace_ingredient` + (`change_recipe_type` 若在范围) 使离群配方对齐同类家族。
- **`differentiate(group)`** = `retag`(拆入不同子 tag/层级) + `rename`(不同变体名) + 配方链整合(`replace`/桥接)。**保留并使区分可见**，与 unify 相反。
- **`constrain_inputs(slot, allow|deny)`** = 收窄（把宽 tag 换成精选列表 / 经 `retag` 调整 tag 成员）或扩充某输入槽候选集。

### ✅已定 · 决策 #4：探测+搁置类（v1 不自动执行）
- **`add_item`**：探测"规范链缺环"（如缺"裹糊虾"），**仅产出搁置建议**；真正生成经 **IDE「数据驱动物品模板」单独立项**（涉及注册/材质/模型，是另一条工程线）。
- **`add_bridge_recipe`** / **`change_recipe_type`** / **`remove_item`**：同样仅探测+搁置建议。
  > 推荐：v1 这四者一律"发现并建议、不动手"。认可否？

## 4. 级联与爆炸半径（引擎在提议高风险动作前必算）

给定目标对象，引擎从知识图谱算出：
- **配方引用**：所有以它为 输入/输出 的配方（`by_graph references/referenced_by`）。
- **tag 连带**：若 `retag`，所有引用该 tag 的配方。
- **世界已放置**：它是否是可在世界放置的方块？是 → **删除/改注册会让现存存档报错/丢方块**（不可逆）。
- **跨 mod**：引用是否横跨多个 mod。

→ 结果写入决策条目的 `blast_radius`，并驱动**风险分级**（§2 路由）。删除/改 tag/跨 mod/世界已放置 → 即使高置信也 `defer`，并随搁置项**展示完整引用清单**。

---

## 5. 冲突策略（绝不静默）

| 情形 | 策略 |
|---|---|
| 生成的配方 id 与现有冲突 | 默认生成确定性新 id + 记录；可选覆盖（需确认） |
| 同一产物有多条配方 | 对全部匹配项作用；若作用不一致/有歧义 → `defer` |
| 删除/改 仍被引用的物品 | **强制 `defer`** + 完整引用清单（永不静默删） |
| 两条决策触碰同一对象 | 引擎检出 → `defer` 冲突，列出两者 |
| 守恒型 1:1 转化遇 `scale` | 识别（等比 + role）并**跳过**，提示作者 |

---

## 6. dry-run / diff 与可逆

- **dry-run**：引擎只算变更集、**不写盘**；emitter 把它渲染成**逐配方/逐文件 before→after diff**——这就是闸门2 的界面。
- **应用**：把变更作为**覆盖产物**写出（datapack/KubeJS），不动源 JAR。
- **可逆**：撤销 = 删除/禁用产物文件。唯一例外是已影响世界状态的（删方块），故那类操作必经搁置+重警告。

## 7. 走查（拿例子的动作过一遍语义）

- 例01 d1 `replace_ingredient{from:小麦,to:面粉}` → §3 原语，低风险、幂等 ✅；d3 `replace{面团→发酵面团}` 触发 §4 级联（孤立 create:dough）→ 生成搁置项 q2 ✅。
- 例02 #3 油的"输入集校正"= `constrain_inputs`（排除脂肪 + 补全合理油）✅；"脂肪→荤油"= `add_bridge_recipe` → 探测+搁置 ✅。
- 例03 "区分低/高纯铜" = `differentiate`；"形近异物按配方语义交用户" = §5 歧义 `defer` ✅。
- 例05 同名大理石 = `rename`(只改 lang，§3) + 配方链整合(`differentiate`) ✅；删装饰被否，未用 remove_item ✅。

→ 五个例子的动作都能落在确定性语义内，无需引擎做 AI 判断。✅
